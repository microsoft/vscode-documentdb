# Step 7.1.5 — Query Playground Name Unification

**PR:** [#553](https://github.com/microsoft/vscode-documentdb/pull/553)

## Summary

Renamed all "scratchpad" / "scrapbook" terminology to **"Query Playground"** across the entire codebase — user-facing strings, internal identifiers, folder names, file names, command IDs, language IDs, and webpack entry points.

## Motivation

The codebase accumulated three terms for the same feature: "scrapbook" (legacy), "scratchpad" (Steps 5–7), and "playground" (preferred user-facing name). This caused confusion in code reviews, documentation, and user-facing strings. A single pass unified everything.

## What Changed

| Aspect         | Before                                     | After                                      |
| -------------- | ------------------------------------------ | ------------------------------------------ |
| Language ID    | `documentdb-scratchpad`                    | `documentdb-playground`                    |
| Command prefix | `command.scratchpad.*`                     | `command.playground.*`                     |
| Source folders | `src/documentdb/scratchpad/`               | `src/documentdb/playground/`               |
| Classes        | `ScratchpadService`, `ScratchpadEvaluator` | `PlaygroundService`, `PlaygroundEvaluator` |
| TS plugin      | `documentdb-scratchpad-ts-plugin`          | `documentdb-playground-ts-plugin`          |
| Worker bundle  | `scratchpadWorker.js`                      | `playgroundWorker.js`                      |
| User-facing    | "DocumentDB Scratchpad"                    | "Query Playground"                         |

### Naming Convention

- **Pascal Case with space** in menus, titles, headers: "Query Playground"
- **Lowercase** in running text/comments: "query playground"
- **Single word** in identifiers: `playground` (e.g., `PlaygroundService`, not `QueryPlaygroundService`)

## Execution

5 atomic commits, each buildable:

1. Rename files/folders via `git mv` (36 files, preserving history)
2. Update `package.json`, grammar, and plugin configuration
3. Rename identifiers, strings, and comments in source files
4. Update webpack, l10n, packages, changelog
5. Fix import paths and clean up remaining references
