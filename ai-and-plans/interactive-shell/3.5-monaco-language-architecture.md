# Step 3.5 — Monaco Language Architecture Decision

**PR:** Decision step — no standalone PR (validated via POC, integrated in [#518](https://github.com/microsoft/vscode-documentdb/pull/518))

## Summary

Evaluated five architecture options for language support in the Collection View's Monaco-based query editors (filter, project, sort). Selected Option E: a custom `documentdb-query` language that reuses the JavaScript Monarch tokenizer without loading the TypeScript worker.

## Problem

The query editors need:

1. **Relaxed quoting** — unquoted keys, single quotes, double quotes — all valid (no red squiggles)
2. **BSON constructors** — `ObjectId("...")`, `ISODate("...")` should highlight as function calls
3. **Clean completions** — only DocumentDB operators, not JavaScript globals (`console`, `Math`, `function`)
4. **Validation** — syntax errors and near-miss typo warnings

Standard `language="json"` rejects unquoted keys. Standard `language="javascript"` floods completions with JS noise and requires the heavy TypeScript worker.

## Options Evaluated

| Option | Approach                                               | Verdict                                                                                |
| ------ | ------------------------------------------------------ | -------------------------------------------------------------------------------------- |
| A      | JSON with relaxed validation                           | Rejected — JSON grammar can't highlight BSON constructors                              |
| B      | JavaScript with global mode shutdown                   | Rejected — global scope conflicts with other JavaScript editors                        |
| C      | Embedded TS worker with TypeScript type definitions    | Rejected — heavy (~400–600 KB), overkill for small query expressions                   |
| D      | Custom Monarch grammar from scratch                    | Rejected — 50–80 lines to maintain, poor edge-case coverage (regex, template literals) |
| **E**  | **Custom language ID + imported JS Monarch tokenizer** | **Selected** — full JS highlighting, clean completion slate, no TS worker              |

Three independent analyses (from different AI evaluation models) all ranked Option E as the top choice.

## Architecture Decision: `documentdb-query` Custom Language

Register a custom Monaco language `"documentdb-query"` that reuses the JavaScript Monarch tokenizer from `monaco-editor/esm/vs/basic-languages/javascript/javascript.js` but does not attach the JS/TS language service.

### What This Provides

- **JavaScript-quality syntax highlighting** — all tokens (strings, numbers, operators, identifiers, regex literals) tokenize correctly
- **Clean completion slate** — no JS globals/keywords noise; only our `CompletionItemProvider` contributes
- **Scoped to query editors** — document editors stay on `language="json"`; the Query Playground uses `language="javascript"` with the full TS service
- **No TS worker loaded** — ~400–600 KB saved; only `editor.worker.js` is loaded

### POC Results

A proof-of-concept validated the approach across 8 criteria:

| Test                                       | Result                                         |
| ------------------------------------------ | ---------------------------------------------- |
| Relaxed quoting (unquoted, single, double) | ✅ Pass                                        |
| BSON constructor highlighting              | ✅ Pass                                        |
| Only DocumentDB completions, no JS noise   | ✅ Pass                                        |
| Dollar-prefix filtering (`$gt`, `$gte`)    | ✅ Pass                                        |
| BSON constructor snippets                  | ⚠️ Partial — `$` prefix fix deferred to Step 4 |
| Project/Sort editors use same language     | ✅ Pass                                        |
| No TS worker loaded                        | ✅ Pass                                        |
| Webpack build succeeds                     | ✅ Pass                                        |

### Import Path Risk

The path `monaco-editor/esm/vs/basic-languages/javascript/javascript.js` has been stable across Monaco versions 0.20 through 0.52+ (6+ years). If it ever changes, the exported Monarch grammar is ~200 lines and can be vendor-copied as a fallback.
