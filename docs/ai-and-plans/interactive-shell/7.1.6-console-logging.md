# Step 7.1.6 — Query Playground Console Logging & Result Display

**PR:** [#559](https://github.com/microsoft/vscode-documentdb/pull/559)

## Summary

Added `console.log()`, `print()`, and `printjson()` support to the Query Playground, and replaced the result display with a custom `TextDocumentContentProvider` for stable, well-formatted result tabs.

## Console Logging

- **Capture mechanism:** `@mongosh`'s evaluation listener (`onPrint` callback on `ShellInstanceState`) captures all `console.log()`, `print()`, and `printjson()` calls during evaluation
- **IPC transport:** Output is sent from the worker thread to the main thread via a new `consoleOutput` message type
- **Display:** A dedicated "DocumentDB Query Playground Output" output channel shows the output without timestamps or prefixes
- **Auto-reveal:** The output panel appears automatically when console output arrives
- **Console hint:** When output is printed, a hint comment is appended to the result document guiding users to the output channel

## Result Display Improvements

Replaced `openReadOnlyContent` (from `@microsoft/vscode-azext-utils`) with a custom `PlaygroundResultProvider`:

- **Stable tab per file** — each `.documentdb` file gets a deterministic result URI; re-running replaces content in the same tab instead of spawning new ones
- **Custom DocumentDB icon** on the result tab via a dedicated `documentdb-playground-result` language with icon contributions
- **JSONC syntax highlighting** — TextMate grammar delegates to `source.json.comments` for proper comment header + JSON content coloring
- **Unified error display** — errors show in the same result tab as successes (no accumulating error tabs)
- **Automatic cleanup** — content removed from provider when result tab is closed

## Help Command Update

Added a "Console Output" section to the help text documenting `console.log()`, `print()`, and `printjson()` support.
